# تشغيل الـ Backend (حل خطأ ERR_CONNECTION_REFUSED)

خطأ **`net::ERR_CONNECTION_REFUSED`** أو **`GET http://localhost:8082/...`** يظهر عندما الواجهة الأمامية تحاول الاتصال بالـ API والـ Backend **غير مشغّل**.

## تشغيل الـ Backend

### الطريقة الأسهل (Windows، حتى لو مسار المشروع فيه أحرف عربية)

من **مستكشف الملفات** افتح مجلد المشروع `mechnow` ثم **انقر نقراً مزدوجاً** على الملف:

- **`start-backend.bat`**

يجب أن تفتح نافذة سوداء (CMD) وتظهر فيها رسالة مثل: `Server running on http://localhost:8082`. **اترك النافذة مفتوحة** طوال فترة التطوير.

### من الطرفية (Terminal)

من مجلد المشروع:

```bash
cd backend
npm install
npm run dev
```

على **Windows** إذا كان المسار فيه عربي وكانت PowerShell تعطي خطأ، استخدم **CMD** بدلاً منها وانتقل لمجلد المشروع ثم نفّذ:

```cmd
cd backend
npm run dev
```

يجب أن تظهر الرسالة:
```text
Server running on http://localhost:8082
```

بعدها الواجهة الأمامية ستستطيع استدعاء:
- `http://localhost:8082/providers/nearby`
- `http://localhost:8082/health`
- وباقي الـ APIs

## إذا لم يبدأ السيرفر

1. **تأكد أن المنفذ 8082 غير مستخدم** من برنامج آخر.
2. **إذا كنت تستخدم قاعدة بيانات (Prisma):**
   ```bash
   cd backend
   npx prisma generate
   ```
   ثم أنشئ ملف `.env` من `.env.example` وضَع `DATABASE_URL` (واختياريًا JWT_SECRET و JWT_REFRESH_SECRET). إذا لم تضف قاعدة بيانات، السيرفر يبدأ برسالة تحذير عن فشل الاتصال بقاعدة البيانات وستعمل على الأقل نقطة `/health`.
3. **تشغيل الواجهة والـ Backend معًا:** شغّل الـ Backend في طرفية أولى (`cd backend && npm run dev`)، ثم الواجهة في طرفية ثانية (`npm run web` أو `npm start`).

## التحقق

- في المتصفح: افتح **http://localhost:8082/health**  
  يجب أن ترى شيئًا مثل: `{"status":"ok","timestamp":"..."}`  
  إذا ظهرت هذه الصفحة فالـ Backend يعمل والواجهة ستستطيع الاتصال به.

## التطبيق على موبايل فعلي (جهازك الحقيقي)

على الجهاز، `localhost` = الموبايل نفسه وليس الكمبيوتر. لذلك:

1. اعرف **IP الكمبيوتر** على الشبكة المحلية (مثلاً من `ipconfig`: عنوان IPv4 مثل `192.168.1.5`).
2. في **جذر المشروع** أنشئ ملف **`.env`** (أو عدّل الموجود) وضَع:
   ```env
   EXPO_PUBLIC_API_URL=http://192.168.1.5:8082
   ```
   (استبدل `192.168.1.5` بـ IP جهازك.)
3. أعد تشغيل الواجهة (`npm start` أو `npm run web`). بعدها التطبيق على الموبايل سيتصل بالـ Backend على الكمبيوتر.
