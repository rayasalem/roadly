# استكشاف أخطاء الويب (Web Troubleshooting)

## أخطاء شائعة في Console

### 1. `AppEntry.bundle` — Failed to load resource: net::ERR_CONNECTION_REFUSED

**السبب:** المتصفح يحاول تحميل حزمة JavaScript من خادم التطوير (Expo/Metro) لكن الخادم لا يعمل أو الصفحة فُتحت قبل تشغيله.

**الحل:**
1. شغّل الواجهة من جذر المشروع:
   ```bash
   npx expo start
   ```
2. انتظر حتى تظهر رسالة مثل: `Metro waiting on ...`
3. اضغط **w** لفتح النسخة الويب، أو افتح يدوياً الرابط الذي يظهر (مثلاً **http://localhost:8081**).
4. لا تفتح التبويب من رابط قديم أو من حفظ سابق قبل تشغيل `expo start`.

---

### 2. `:8081/favicon.ico` — Failed to load resource: net::ERR_CONNECTION_REFUSED

**السبب:** عنوان طلب الـ favicon غير صحيح (يظهر كـ `:8081/favicon.ico` بدون النطاق)، فيحدث الطلب إلى عنوان خاطئ فتفشل الاتصال.

**الحل:**
1. افتح التطبيق دائماً بالعنوان الكامل، وليس فقط `:8081`:
   - ✅ **http://localhost:8081**
   - ✅ **http://127.0.0.1:8081**
   - ❌ تجنّب إدخال `:8081` فقط في شريط العنوان.
2. (اختياري) لإظهار أيقونة التبويب وتقليل الطلبات الفاشلة، ضع ملف favicon في المجلد `public/`:
   - انسخ أيقونة التطبيق (مثلاً من `assets/brand/`) إلى `public/favicon.ico`، أو
   - أنشئ مجلد `public` في جذر المشروع وضَع فيه `favicon.ico`.

---

## ملخص سريع

| الخطأ | الإجراء |
|--------|---------|
| ERR_CONNECTION_REFUSED على bundle | تشغيل `npx expo start` ثم فتح الرابط المعروض (مثلاً http://localhost:8081) |
| favicon على :8081 | فتح التطبيق بـ http://localhost:8081 وإضافة `public/favicon.ico` إن أردت |

بعد تطبيق ذلك، أعد تحميل الصفحة (F5) أو أعد تشغيل Expo ثم افتح الرابط من جديد.
