# 0. ناف بار (Navigation Bar) — لوحة المزود

## 0.1 المواصفات
- **يظهر في كل صفحة** لوحة تحكم المزود، قابل للتنقل بسهولة.
- **موضع:** على الموبايل شريط أفقي **أسفل الشاشة**؛ على الشاشات الواسعة (≥600px) شريط **جانبي أيسر**.
- **تمييز القسم النشط:** لون مختلف (أخضر #22C55E للنشط، رمادي للنوع غير النشط).
- **أيقونات واضحة** لكل قسم؛ نصوص قصيرة تحت الأيقونة (أو بجانبها في السايدبار).

## 0.2 الأقسام الثمانية (كل قسم له وظيفة محددة)

| # | القسم (عربي) | القسم (EN) | الأيقونة (اقتراح) | الوظيفة |
|---|--------------|------------|-------------------|----------|
| 1 | الرئيسية | Dashboard | view-dashboard-outline | عرض ملخص: توافر + طلبات جديدة + جارية + مكتملة في صفحة واحدة |
| 2 | الطلبات الجديدة | Incoming Requests | bell-alert-outline | قائمة الطلبات الجديدة/قيد الانتظار فقط + أزرار قبول/رفض |
| 3 | الطلبات الجارية | In Progress | progress-clock | قائمة الطلبات قيد التنفيذ فقط + "عرض على الخريطة" + "تم التنفيذ" |
| 4 | الطلبات المكتملة | History | history | سجل الطلبات المكتملة مع التاريخ والعميل ونوع الخدمة |
| 5 | الخريطة | Map | map-marker-radius | فتح شاشة الخريطة لعرض مواقع الطلبات والعملاء في الوقت الحقيقي |
| 6 | الإحصائيات | Stats | chart-bar | أعداد الطلبات (يوم/أسبوع/شهر)، الأرباح، معدل القبول، متوسط التقييم، رسم بياني |
| 7 | الملف والإعدادات | Profile & Settings | account-cog-outline | الانتقال إلى الملف الشخصي والإعدادات (تعديل البيانات، إشعارات، لغة) |
| 8 | المساعدة | Help | help-circle-outline | نصوص المساعدة والدعم الفني وطرق التواصل |

## 0.3 التصرف على الهاتف والويب
- **هاتف:** شريط سفلي أفقي؛ يمكن عرض الأقسام في صف واحد (أيقونة + نص قصير) أو تمرير أفقي إن لزم.
- **ويب/تابلت:** سايدبار ثابت بعرض ~200px؛ نفس الأقسام الثمانية عمودياً مع أيقونة + نص.

## 1.2 الإحصائيات (قسم Statistics)
| الحقل | الوصف |
|-------|--------|
| طلبات اليوم | عدد الطلبات اليوم |
| طلبات الأسبوع | عدد الطلبات أسبوعياً |
| طلبات الشهر | عدد الطلبات شهرياً |
| إجمالي الأرباح | مبلغ (مثلاً د.أ) |
| معدل القبول | نسبة مئوية |
| **متوسط التقييم** | ★★★★☆ + رقم (مثلاً 4.5) |

## 1.3 الإشعارات والتوافر
- **إشعارات فورية:** عند وصول طلب جديد أو تحديث حالة → Toast + (اختياري) صوت/اهتزاز.
- **عداد الإشعارات:** على أيقونة الجرس في الهيدر (عدد الطلبات الجديدة أو غير المقروءة).
- **زر التوافر:** واضح، لون أخضر (متاح) أو أحمر (غير متاح)، يغيّر الحالة فوراً.

## 1.4 الخريطة للمزود
- زر **"عرض على الخريطة"** في كل طلب جاري → يفتح شاشة الخريطة (Map) لرؤية موقع العميل/الطلب.
- (مستقبلاً: خريطة مخصصة للمزود تعرض مواقع الطلبات والعملاء في الوقت الحقيقي.)

# 2. تحسين عرض الخريطة (Map)

## 2.1 أيقونات المزودين حسب المهنة
| نوع المزود | الأيقونة | اللون (خلفية الماركر) |
|------------|----------|------------------------|
| ميكانيكي | مفتاح (wrench) | أزرق #2563EB |
| ونش | ونش (tow-truck) | أحمر #DC2626 |
| مؤجر سيارات | سيارة (car-side) | أخضر #22C55E |

- الحالة: متاح = لون النوع، مشغول = برتقالي، غير متصل = رمادي.

## 2.2 البطاقة المنبثقة (Popup) عند الضغط على الماركر
```
┌──────────────────────────────┐
│ اسم المزود                   │
│ 🔧 نوع الخدمة (ميكانيكي)     │
│ ● متاح / غير متاح            │
│ ★ 4.5 · 2.1 km               │
│ [طلب الخدمة]                 │
└──────────────────────────────┘
```

## 2.3 التصرف والمسافات
- **تحديث المواقع:** جلب دوري (refetch) أو WebSocket للوقت الحقيقي.
- **تجنب التراكب:** تجميع (Clustering) عند زيادة عدد الماركرات في منطقة واحدة.
- **أيقونات:** حجم مناسب للضغط (~48px)، حدود بيضاء، ظل خفيف.

## 2.4 الفلترة على الخريطة
- فلتر حسب **نوع الخدمة:** الكل / ميكانيكي / ونش / تأجير.
- (اختياري) فلتر **تقييم:** مثلاً 4+ نجوم عند دعم الـ API.

# 3. إدارة حالة الطلبات (Request Status)

## 3.1 الحالات والألوان
| الحالة | المفتاح | اللون (تقريبي) |
|--------|---------|----------------|
| جديد | new | رمادي فاتح #9CA3AF |
| قيد الانتظار | pending | رمادي فاتح #9CA3AF |
| مقبول | accepted | أخضر #22C55E |
| مرفوض | rejected | أحمر #DC2626 |
| جاري التنفيذ | in_progress | أزرق #0284C7 |
| مكتمل | completed | ذهبي/برتقالي #F59E0B |
| ملغى | cancelled | رمادي غامق #374151 |

## 3.2 عرض الطلب للعميل
- **بطاقة طلب:** اسم المزود، نوع الخدمة، موقع (إحداثيات أو رابط خريطة)، وقت الطلب، **شارة الحالة** بلون الحالة.
- **أزرار:** تتبع على الخريطة (للطلبات الجارية)، تقييم المزود (للمكتملة)، إلغاء الطلب (لجديد/قيد الانتظار فقط).
- **تحديث فوري:** إعادة جلب دورية أو إبطال الاستعلام عند تحديث الحالة.

## 3.3 عرض الطلب للمزود
- أقسام: طلبات جديدة (قبول/رفض)، جاري التنفيذ (عرض على الخريطة + تم التنفيذ)، مكتملة.
- بعد القبول → الحالة تنتقل إلى مقبول/جاري التنفيذ؛ بعد "تم التنفيذ" → مكتمل مع إشعار للعميل.

# 4. نظام التقييم بعد انتهاء الخدمة

## 4.1 عناصر التقييم (صفحة التقييم)
| العنصر | النوع | إلزامي |
|--------|--------|--------|
| التقييم العام | نجوم 1–5 | نعم |
| سرعة الاستجابة | نجوم 1–5 | لا |
| جودة الخدمة | نجوم 1–5 | لا |
| سلوك المزود / الاحترافية | نجوم 1–5 | لا |
| تعليق نصي | حقل متعدد الأسطر | لا |

- ألوان النجوم: أصفر/ذهبي #F59E0B.

## 4.2 بعد الإرسال
- تأكيد: **"شكراً لتقييمك"** مع زر "العودة لطلباتي".

## 4.3 للمزود
- **لوحة التحكم:** قسم "التقييمات الواردة" يعرض كل تقييم (عميل، نجوم عامة، سرعة/جودة/احترافية، تعليق، تاريخ).
- **الإحصائيات:** متوسط التقييم (نجوم + رقم).
- **على الخريطة:** معدل تقييم المزود يظهر في الـ Popup أو تحت اسم المزود (مثلاً ★ 4.5).
- **إشعار:** عند وصول تقييم جديد (نوع إشعار من الباكند أو محلي).

# 5. تحسين تجربة المستخدم (UX)

## 5.1 فلترة المزودين على الخريطة
- حسب **النوع:** الكل، ميكانيكي، ونش، تأجير.
- (مستقبلاً) حسب **التقييم** (مثلاً 4+)، **قرب الموقع** (من الـ API)، **سرعة الاستجابة**.

## 5.2 المزودون المفضلون
- زر/قسم **"المفضلة"** في لوحة العميل للوصول السريع لمزودين محفوظين.
- (تنفيذ: قائمة مفضلين محلية أو من الباكند + شاشة Favorites.)

## 5.3 طلب مسبق (الوقت المفضل)
- في نموذج إنشاء الطلب: حقل **"الوقت المفضل"** (مثلاً "في أقرب وقت" أو "2:00 م") — موجود في QuickRequestForm.

## 5.4 متابعة الطلب على الخريطة (Real-time Tracking)
- زر **"تتبع على الخريطة"** في الطلبات الجارية → شاشة LiveTracking تعرض المسار والموقع.
- تحديث الموقع: جلب دوري أو WebSocket.

## 5.5 إلغاء وتعديل الطلب
- **إلغاء:** متاح للعميل عندما تكون الحالة جديد أو قيد الانتظار (زر "إلغاء الطلب").
- **تعديل:** (مستقبلاً) زر تعديل في تفاصيل الطلب قبل أن يقبله المزود.

# 6. الإشعارات الذكية

## 6.1 للمزود
| الحدث | نص الإشعار (مثال) |
|-------|-------------------|
| طلب جديد | طلب خدمة جديد — أحمد، صيانة |

## 6.2 للعميل
| الحدث | نص الإشعار (مثال) |
|-------|-------------------|

| رفض الطلب | تم رفض طلبك |

  - إتمام الخدمة والتقييم
- إشعارات صوتية أو اهتزاز عند وصول طلب جديد (اختياري، مثلاً عبر expo-haptics).

# 8. التصميم البصري

## 8.1 المبادئ
- **خلفية فاتحة** للشاشات الرئيسية (#F3F4F6 أو ما شابه).
- **ألوان متناسقة:** أخضر للعمليات الإيجابية، أحمر للرفض/الإلغاء، أزرق للمعلومات والجاري.
- **نصوص كبيرة** وقابلة للقراءة على الهاتف (عنوان 18–20، body 15–16، caption 12–13).
- **أيقونات واضحة** لكل قسم (طلبات، خريطة، إحصائيات، تقييمات، إشعارات، إعدادات).

## 8.2 ترتيب الأقسام المنطقي
1. **لوحة التحكم** (رئيسية للعميل أو المزود).
2. **الطلبات** (سجل الطلبات، فلترة، تفاصيل، إجراءات).
3. **الخريطة** (مزودون، فلترة، Popup، طلب خدمة).
4. **الإحصائيات** (للمزود: أعداد، أرباح، تقييم).
5. **التقييمات** (صفحة التقييم للعميل؛ التقييمات الواردة + متوسط للمزود).
6. **الإشعارات** (قائمة بالإشعارات مع تاريخ ووقت ونوع).
7. **الملف الشخصي والإعدادات** (حساب، إشعارات، لغة، تسجيل خروج).

## 8.3 الأزرار والبطاقات
- **أزرار أساسية:** لون مميز (أخضر أو أزرق)، ارتفاع مريح (~48px)، نص واضح.
- **بطاقات:** خلفية بيضاء، ظل خفيف، حواف مستديرة (12–16px).

| **قائمة الإشعارات** | عرض كل الإشعارات مع النوع والتاريخ؛ الضغط → الانتقال للتفاصيل |
| **إعدادات الإشعارات** | تشغيل/إيقاف أنواع الإشعارات (طلبات جديدة، تحديثات حالة، إكمال وتقييم) |
